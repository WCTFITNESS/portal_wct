================================================================================
  Portal WCT no Render.com (Docker — igual ideia do Tracking)
================================================================================

1) No Render: New + > Web Service > conecte o repo portal_wct.

2) Em "Runtime" escolha DOCKER (nao Native Node).

3) Dockerfile Path:  Dockerfile   (na raiz do repositorio)
   Docker Build Context Directory: vazio ou .

4) Plano: Starter ou superior (Free pode limitar Docker; se falhar, use Starter).

5) Variaveis de ambiente (Environment) — obrigatorias para o app subir:

   PORTAL_BASE_URL=/
   PORTAL_TZ=America/Sao_Paulo

   Banco (PostgreSQL recomendado no Render):

   Opcao A — URL unica (recomendado):
   PORTAL_DATABASE_URL=postgresql://usuario:senha@HOST:5432/nome_do_banco
   (tambem aceita mysql://... para ambientes legados)

   No Render, ao vincular um Postgres ao Web Service, o painel pode injetar
   DATABASE_URL=postgresql://... — o Portal le essa variavel automaticamente.

   Se ainda existir PORTAL_DATABASE_URL antiga em mysql:// e o servico tiver
   DATABASE_URL do Postgres (Render injeta ao vincular o banco), o app prefere
   Postgres. O ideal e apagar URL mysql e credenciais Railway do ambiente.

   Opcao B — variaveis separadas:
   PORTAL_DB_DRIVER=pgsql
   PORTAL_DB_HOST=<hostname real do postgres>
   PORTAL_DB_PORT=5432
   PORTAL_DB_NAME=<nome do banco>
   PORTAL_DB_USER=<usuario>
   PORTAL_DB_PASS=<senha>

   TRACKING_PUBLIC_URL=https://SEU-TRACKING.onrender.com/admin/dashboard

   O Render define PORT sozinho. O entrypoint alinha Listen E <VirtualHost *:PORT>
   (só mudar Listen sem o VirtualHost costuma dar 403 Forbidden no Render).

   Se aparecer erro "getaddrinfo for mysql failed": o Portal ainda esta usando host
   padrao "mysql". Defina PORTAL_DATABASE_URL (ou PORTAL_DB_* + PORTAL_DB_DRIVER); o
   docker-entrypoint gera config/db-runtime.php pelo PHP CLI (que sempre le as vars).

6) Schema automatico:
   - Para PostgreSQL, o startup aplica database.pgsql.sql automaticamente.
   - Para MySQL, o startup aplica database.sql automaticamente.
   Controle por PORTAL_AUTO_MIGRATE=1 (padrao) e PORTAL_SCHEMA_FILE (opcional).

7) Health Check Path sugerido: /index.php?page=dashboard

8) Opcional: use o arquivo render.yaml na raiz como Blueprint (ajuste nome/regiao).
